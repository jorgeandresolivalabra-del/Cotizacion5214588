Propuesta general: Se propone el desarrollo de un sistema completo de pedidos de almuerzo diseñado como una pequeña plataforma SaaS con lógica de negocio real. El sistema permite a los usuarios seleccionar su almuerzo desde un menú dinámico con precios, pagar en línea, y recibir confirmación automática tras el pago. Incluye backend con Next.js API Routes, Supabase (PostgreSQL), lógica de corte a las 20:00 hrs, webhooks de MercadoPago y panel de administración. Opcionalmente, los pedidos se exportan a Google Sheets en tiempo real.
Alcance del proyecto — Módulos incluidos
  • 1. Menú Dinámico con Precios y Categorías
    • Menú configurable por día con platillos, precios y categorías.
    • Activación/desactivación de ítems desde el panel de administración sin tocar código.
    • Imágenes opcionales por platillo almacenadas en Supabase Storage.
    • Vista de usuario clara, rápida y optimizada para dispositivos móviles.
  • 2. Sistema de Pedidos + Base de Datos Supabase
    • Flujo completo: selección de platillo → pedido → pago → confirmación automática.
    • Almacenamiento en Supabase (PostgreSQL) con esquema relacional optimizado.
    • Row Level Security (RLS) para que cada usuario solo vea sus propios pedidos.
    • Exportación opcional a Google Sheets en tiempo real vía Google Sheets API.
  • 3. Integración de Pagos — MercadoPago
    • Integración con la API oficial de MercadoPago para pagos en línea.
    • Confirmación automática de pago vía webhook procesado en el servidor.
    • Actualización del estado del pedido en la base de datos al recibir confirmación.
    • Historial de transacciones por usuario con estado visible en su perfil.
  • 4. Corte Automático de Pedidos a las 8 PM (Lógica de Backend)
    • Validación en el servidor (Next.js API Routes) que rechaza pedidos fuera de horario.
    • El sistema se cierra automáticamente a las 8 PM sin intervención manual.
    • Mensaje de estado claro al usuario cuando el sistema está fuera de horario.
    • Hora de corte configurable por el administrador desde el panel, sin cambiar código.
  • 5. Panel de Administración (Pedidos y Producción)
    • Vista de pedidos del día ordenados por estado: pendiente, pagado, en preparación, entregado.
    • Panel de producción para cocina con actualización de estado en tiempo real (Supabase Realtime).
    • Gestión completa del menú: agregar, editar, activar o desactivar platillos.
    • Resumen diario de ventas, totales por platillo e ingresos del día.
  • 6. Autenticación y Roles (Supabase Auth)
    • Autenticación segura vía Supabase Auth (email/password o magic link).
    • Roles diferenciados: usuario final para hacer pedidos y administrador para el panel.
    • Rutas protegidas en Next.js con middleware de sesión del lado del servidor.
    • Registro simplificado para usuarios habituales del sistema.
  • 7. Despliegue, Documentación y Entrega
    • Despliegue en Vercel con dominio propio, SSL y CI/CD automático desde GitHub.
    • Variables de entorno seguras para tokens de pago, claves de Supabase y Google.
    • Documentación técnica de uso del sistema y del panel de administración.
    • Repositorio privado en GitHub entregado al cliente al completar el pago.
